Running the application

  1. PREPARE
    1. ACTIVATE THE VIRTUAL ROBOT SERVER
    2. ACTIVATE THE MQTT BROKER (mosquitto)

  2. ACTIVATE THE ROBOT BASIC-MIND
    1. RUN MainCtxRobotMain IN it.unibo.eclipse.qak.robotMinds19
    2. RUN MainCtxRobotAppl IN it.unibo.eclipse.qak.robotMinds19 (just a test)
      LOOK AT logFiltered.txt (the walls found as obstacles)

  3. BUSINESS LOGIC
    1. RUN MainCtxBoundaryPlanned IN it.unibo.eclipse.qak.planning19 => LOOK AT roomMap.txt (the map of the room)
    2. RUN MainCtxRoomExplore IN it.unibo.eclipse.qak.planning19 => LOOK AT roomMapWithTable.txt (the map of the room with the table)
    3. RUN MainCtxWorkInRoom IN it.unibo.eclipse.qak.planning19 => The robot goes FROM (0,0) to (5,3)

  4. M2M INTERACTION
    1. RUN MainCoapClientModelRobot IN it.unibo.eclipse.qak.robotMinds19
    2. RUN resourceObserverCoapClient IN it.unibo.eclipse.qak.robotMinds19

  5. HUMAN-MACHINE INTERACTION (front end)
    1. RUN node frontendServer IN it.unibo.frontend19/nodeCode/frontend



Scenarios (incremental design and development)

1

roomboundaryplanned.qak

Builds a map of the empty room by using the planner.

2

roomexplore.qak

If no map exists, explore the room boundary by using the planner;
afterwards, finds the position of the table

3

workinroom.qak

The robot plans a trip from home to the east part of the table and executes it.
