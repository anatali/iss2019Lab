 The Web server handles the command sent by the user and then puts in execution the 'business logic' by sending (via MQTT) :
  1. the qak dispatch robotCmd to the robotmind, or
  2. the qak dispatch modelChange to the resourcemodel
In the second interaction mode, all the actions, including the execution of the business logic by the 'mind', are a consequence of a change in the resource model.

The function delegate

The function delegate in applCode.pl can do one of these possible actions:
  1. publishMsgToRobotmind(cmd):
  2. publishEmitUserCmd(cmd)
  3. publishMsgToResourceModel(cmd)
  4. changeResourceModelCoap(cmd)

The system model:robotmind.qak

The resource model:resourceModel.pl

Machine-to-machine interaction using CoAP

 The Web server handles the command sent by the user and then puts in execution the 'business logic' by sending :
  1. (via MQTT): the qak dispatch modelChange to the resourcemodel
  2. (via CoAP): a PUT request to change the resourcemodel
In the CoAP-based interaction mode, the resource model, is accesses in a standard, RESTful way.
