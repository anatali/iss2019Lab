A button on the RaspberryPi

A button is an input device that, like any input device on Raspberry, MUST have a terminale connected to a 3.3V pin (and NOT to a 5V pin).

The other terminal should be connected to a pin used as input, e.g. the GPIO-24 pin. To avoid floating connections (and random input values) it is appropiate to connect the input side of the button to the GND pin through a resistor of 10K ohm (marron, black, orange) in order to allow the current to flow (when the button is pressed) along a path with lower resistance via the input pin GPIO-24 .

Modelling a button (domain model)

  • For a software deisgner, a physical button (as any other any input device) is always associated with some basic software (usually a driver).
  • A driver usually provides the button state via polling or via interrupt.
  • Some higher-level library (e.g. Pi4J) can provide a button as an observable (in the sense of GOF design pattern) entity
