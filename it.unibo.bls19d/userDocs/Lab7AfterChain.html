72939 - LAB7AfterChain | The impact of DSL and (meta)models

LabISS-lectures site

Overview

  1. Logic first: technology aware but technology agnostic. We could start from a model of the logical architecture of the system.

  2. The model of the logical architecture of a software should describe essentials aspects about structure, interaction and behaviour of the system starting from the requirements.

  3. Any model is based on a meta-model (see Metamodeling).

  4. In the context of agile development processes like SCRUM (wikipedia) (SCRUM guide), a model of the logical architecture of the system can be useful to organize the work of the team.
    • Model-driven engineering
    • Model-driven architecture

  5. An executable model is supposed to increase ease of reuse and to lower the cost of software development (see for example Executable UML).

  6. Executable models can be written by introducing an (Internal and External) Domain-specific language
    • What is a Domain Specific Language? (Fowler)
    • Domain-specific modeling

  7. Kotlin provides support for writing internal DSL (see LABDsl | Kotlin Internal DSL).

  8. An external DSL could help the work of application designers. Let us experiment with the custom QakDSL built by using the Xtext Technology. See the project it.unibo.eclipse.qak.chainPlus
