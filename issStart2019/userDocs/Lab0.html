START: GRADLE

  1. Download gradle (using zip or SDKMAN). Current version: 5.2.1.
  2. Set GRADLE_HOME to the unpacked Gradle files and add GRADLE_HOME/bin to your PATH environment variable
  3. Create a work dir (e.g. issStart2019) and work in it
  4. Execute gradle wrapper (see https://docs.gradle.org/current/userguide/gradle_wrapper.html)
  5. Execute gradlew init and select 8 (kotlin-application), 1 (groovy)
  6. Look at the generated file AppK.kt
  7. Execute gradlew run. The resullt is: 
    Task :run
Hello world.

START: A JAVA APP

  1. Add the folder src/main/java
  2. Add the class AppJava.java in src/main/java/issStart2019
  3. In build.gradle, set mainClassName='issStart2019.AppJava'
  4. Execute gradlew build
  5. Look at the directory build, and execute gradlew run. The result is: 
    Hello world FROM JAVA.

FROM Java TO Kotlin USING IntelliJ IDEA

  1. Copy AppJava.java in a new class named AppKtFromJava.java
  2. Open the current working folder (mpuse on it) as IntelliJ IDEA Community Edition Project.
  3. Select AppJava.java and click on Convert Java file to Kotlin file (CTRL-ALT-SHIFT-K) and compare the generated AppKtFromJava.kt with the original AppK.kt
  4. In build.gradle, set mainClassName='it.unibo.bls19.AppKtFromJava'
  5. Execute gradlew run

GIT : starting

There are many info sources; for example: GIT Command Line Basic (video)c and Basic Git commands.
  1. Execute git init in the dir issStart2019.
  2. Execute git status.
  3. Edit the file .gitignore
  4. Execute git log.
  5. Execute git add -A and git status.
  6. Execute git reset to undo staging.
  7. Excute git config --global user.name "Your Name", git config --global user.email "Your Email"
  8. Execute git config --list.
  9. Execute git commit -m "First" and git status.
  10. Execute git log.
  11. Execute git rm file.txt (--cached file.txt) to delete (remove) files.
  12. Execute git remote add origin https://github.com/... to connect to a remote GIT.
  13. Execute git checkout master git merge origin/master --allow-unrelated-histories if needed.
  14. Execute git push origin master to update the remote GIT

GIT : clone the iss2019Lab

  1. Open a new, empty directory (e.g. iss2019Lab) and work in it.
  2. Execute git clone https://github.com/anatali/iss2019Lab.git
  3. Execute git remote -v
  4. Use the files as a read-only reference: never change them
  5. Execute git pull when needed

GIT : useful commands 

  • git config --global core.autocrlf input
    Convert all CRLF line endings to LF before it stores it in the commit.
    When we clone a project that was created on the MS Windows platform MS Windows typically put append a hidden CR + LF characters on the end of each line of a text file. 
  • git status --h
  • git status --ignore
  • git rm --cached build -r
    REMOVE from Staging area
